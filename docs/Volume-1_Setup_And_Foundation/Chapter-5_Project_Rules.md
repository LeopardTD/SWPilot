# Chapter 5 — Project Rules

## Tujuan

Menetapkan standar pengembangan agar kode tetap konsisten selama proyek berlangsung.

---

# Coding Rules

- Business Logic dipisahkan dari UI.
- Tidak menggunakan magic number.
- Tidak melakukan hardcode warna.
- Tidak melakukan hardcode padding.
- Satu widget satu file (jika memungkinkan).
- Struktur folder harus konsisten.

---

# Documentation Rules

Setiap fitur harus memiliki:

- Prompt
- Screenshot
- Checklist
- Changelog

---

# Git Rules

Gunakan Conventional Commit.

Contoh:

```text
feat(ui): create dashboard

fix(theme): fix dark mode

docs(volume1): update setup guide
```
